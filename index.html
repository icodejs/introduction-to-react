What is React?

Its a JavaScript Library for building User Interfaces created by Facebook

  • It only focuses on the view layer (V in MVC)
  • Makes no assumptions about the rest of your technology stack
  • Can be plugged into an existing MVC framework e.g.
    • Directives in AngularJS
    • Components in Ember
    • Backbone item views
  • This comes in 3 flavours
    • React JS (Web)
    • React Native (iOS)
    • React Native (Android)

What is MVC?

Stands for Model, View, Controller

Software architectural pattern for implementing user interfaces

Model: Model is where the application’s data objects are stored. The model doesn’t know anything about views and controllers.

View: View is what's presented to the users and how users interact with the app.

Controller: The controller is the decision maker and the glue between the model and view. The controller updates the view when the model changes. It also adds event listeners to the view and updates the model when the user manipulates the view.

Misconception (What React isn't)

  • Its a front-end framework
  • Its an alternative to Angular, Ember, Backbone
  • Its template library (like: handlebars.js, mustache.js)

How do I structure my app?

Flux (Created by Facebook)

An application architecture based on a unidirectional data flow.

Event and data flow

Dispatcher

  • Pub sub system using Node’s Event Emitter
  • Receives actions from dispatchers
  • Broadcasts the payload to ALL of its registered callbacks (Stores)
  • Has no knowledge about any stores
  • There is only ever one Dispatcher, and it acts as the central hub within your application

Actions (Not DOM events)

  • Not elaborate setters
  • Represent mutations in you application state
  • Think of actions as Newspapers, reporting on something that happened in the world / App
  • E.g. Like a new feed added to feed page
  • Follow another user
  • Post a comment

Stores (Model like but not)

  • Read-only
  • Self contained universe
  • Updated as a result of actions
  • Register callbacks with the app dispatcher
  • Data is updated via the app dispatcher events not setters

Views

  • This is where React kicks in
  • Controller views listen and react to changes within stores
  • Update their state and re-render themselves and their child views
  • Respond to UI interaction and fire actions that can mutate stores

Virtual DOM (HOW AND WHY)

  • Incredibly fast
  • It utilises an in-memory DOM
  • Which means a copy of your DOM is kept in memory and any changes are made in memory and updated in the DOM when needed
  • Events are normalised across browsers and automatically delegated

JSX (In-line HTML/XML - WTF!)

  • Components can be written in pure JavaScript or what Facebook calls JSX, a JavaScript XML syntax
  • Allows you to write HTML inside of JavaScript without having to wrap it in quotes
  • We've all been conditioned to separate our functionality from our view, but if you think about it, they are actually intrinsic tied to one another
