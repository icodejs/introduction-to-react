What is React?

Its a JavaScript Library for building User Interfaces created by Facebook

  • It only focuses on the view layer (V in MVC)
  • Makes no assumptions about the rest of your technology stack
  • Can be plugged into an existing MVC framework e.g.
    • Directives in AngularJS
    • Components in Ember
    • Backbone item views
  • This comes in 3 flavours
    • React JS (Web)
    • React Native (iOS)
    • React Native (Android)

What is MVC?

Stands for Model, View, Controller

Software architectural pattern for implementing user interfaces

Model: Is where the application’s data objects are stored. The model doesn’t know anything about views and controllers.

View: Is what's presented to the users and is how users interact with the app.

Controller: Is the decision maker and the glue between the model and view. The controller updates the view when the model changes. It also adds event listeners to the view and updates the model when the user manipulates the view.

Misconception (What React isn't)

  • Not a front-end framework
  • Not a straight alternative to Angular, Ember, Backbone
  • Not a template library (like: handlebars.js, mustache.js)

How do I structure my app?

Flux (Created by Facebook)

An application architecture based on unidirectional data flow.

Event and data flow

Dispatcher

  • Pub sub system using Node’s Event Emitter
  • Broadcasts its payload to ALL of its registered callbacks (Stores)
  • Has no knowledge about any stores
  • There is only ever one Dispatcher, and it acts as the central hub within your application

Actions (Not DOM events)

  • Not elaborate setters
  • Represent mutations in you application state
  • Think of actions as Newspapers, reporting on something that happened in the world / App
  • E.g.
    • A new feed added to feed page
    • Follow another user
    • Post a comment

Stores (Model like... but not)

  • Read-only
  • Self contained universe
  • Updated as a result of actions
  • Register callbacks with the Dispatcher
  • State is updated via the Dispatcher events, NOT SETTERS
  • Events are emitted when internal data changes

Views

  • This is where React gets to shine
  • Controller views listen and react to store events
  • They can optionally update their state, re-render themselves and their child views
  • Also, they can respond to user interaction and fire actions that will eventually mutate Stores

Virtual DOM (HOW AND WHY)

  • Incredibly fast
  • Utilises an in-memory DOM
  • Re-rendering the virtual DOM is very cheap, and updates to the real DOM are localised, only to the parts that have changed
  • Events are normalised across browsers and automatically delegated

JSX (In-line HTML/XML - WTF!)

  • Components can be written in pure JavaScript or what Facebook calls JSX, a JavaScript XML syntax
  • Allows you to write HTML inside of JavaScript without having to wrap it in quotes
  • We've all been conditioned to separate our functionality from our view, but if you think about it, they are actually intrinsically tied to one another
  • Markup, and the code that generates it are intimately connected
  • Display logic is often very complex and using template languages to express it becomes cumbersome

Components

A reusable UI element with a well defined interface.

E.g. Buttons, Form fields, Feed, Tweet, Layout components, Image upload, Auto suggest input.

  • Encourages reuse and abstraction
  • Similar to jQuery plugins
  • Can be as simple as plain HTML
  • Or as complex as HTML, Styles, JS
  • Can contain child components
  • E.g. React Bootstrap

How do I traverse the DOM / Virtual DOM

React encourages the creation of small self-contained components. If you're attempting to traverse the DOM, outside of your component, you're probably doing something wrong!

  • HTML elements within a component can be accessed via ref attributes:
  • Actual DOM nodes are accessed using the React findDOMNode method: 
    
var domNode = React.findDOMNode(this.refs.myTextInput);
  • Once the component has loaded into the DOM, jQuery can be used as normal: 
    
componentDidMount() {
    var $ = require('jquery');
    var $component = $(React.findDOMNode(this));

    var $body = $component.closest('body');
    $body.css({ backgroundColor: '#000' });

    var $input = $component.find('input[ref="myTextInput"]');
    $input.fadeOut(2000);
}

Browser compatibility

  • IE8 +
  • React is pretty good at abstracting browser differences
  • Older browsers may need polyfils
  • Although, some browsers are limited or present quirky behaviors

Server side rendering

(Isomorphic JavaScript)

  • CommonJS
  • Node
  • Browserify (Client bundle.js)
  • Share the same code
  • Increase code reuse
  • Easily done via Node express
  • Server-side solutions available in backend languages (Rails, Scala/Java, PHP)
